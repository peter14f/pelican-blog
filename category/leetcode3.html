public String tree2str(TreeNode t) {
    StringBuilder sb = new StringBuilder();

    if (t != null) {
        sb.append(t.val);
        if (t.left != null) {
            preOrderAppend(sb, t.left);
        } else if (t.right != null) {
            sb.append("()");
        }
        preOrderAppend(sb, t.right);
    }
    return sb.toString();
}

private void preOrderAppend(StringBuilder sb, TreeNode node) {
    if (node == null) {
        return;
    }

    sb.append('(');
    sb.append(node.val);

    if (node.left != null) {
        preOrderAppend(sb, node.left);
    } else if (node.right != null) {
        sb.append("()");
    }
    preOrderAppend(sb, node.right);

    sb.append(')');
}

The recursive solution is not hard to come up, but it takes exponential time.

public int findTargetSumWays(int[] nums, int S) {
    // 2 ^ n total possible choices
    return numWaysSumToTarget(nums, 0, 0, S);
}

private int numWaysSumToTarget(int[] nums, int i, int curSum, int target) {
    if (i >= nums.length) {
        if (curSum == target) {
            return 1;
        } else {
            return 0;
        }
    } else {
        int a = numWaysSumToTarget(nums, i+1, curSum+nums[i], target);
        int b = numWaysSumToTarget(nums, i+1, curSum-nums[i], target);
        return a + b;
    }
}

493-reverse_pairs

Mar 17 2018 Tags Merge Sort / Divide And Conquer / BIT / Self-Balancing BST

LeetCode Problem Link

This problem asks us to find the number of distinct (i, j) pairs such that - i < j - nums[i] > 2*nums[j]

The straightforward solution takes O(n^2) time.

public int reversePairs(int[] nums) {
    int n = nums.length;
    int ans = 0;
    for (int i=0; i<n; i++) {
        for (int j=i+1; j<n; j++) {
            long numI = nums[i];
            long numJ = nums[j];
            if (numI > 2*numJ) {
                ans++;
            }
        }
    }
    return ans;
}
