The problem is very simple. Just tests if you understand how the game works. Check each row, and then check each column. Finally, check each subboard.

public static final int SIZE = 9;
public static final int SUBSIZE = 3;

public boolean isValidSudoku(char[][] board) {

    for (int row=0; row<SIZE; row++) {
        if (!rowIsValid(board, row)) {
            return false;
        }
    }

    for (int col=0; col<SIZE; col++) {
        if (!colIsValid(board, col)) {
            return false;
        }
    }

    for (int row=0; row < SIZE; row+=SUBSIZE) {
        for (int col=0; col < SIZE; col+=SUBSIZE) {
            if (!subboardIsValid(board, row, col)) {
                return false;
            }
        }
    }
    return true;

035-search_insert_position

Sep 27 2015 Tags Binary Search

LeetCode Problem Link

Use binary search. If the number is not found, the l pointer actually will be the correct index if the number were to be inserted.

public int searchInsert(int[] nums, int target) {

    int l = 0;
    int h = nums.length - 1;

    while (l <= h) {

        int middle = l + (h-l)/2;
        int middleNum = nums[middle];

        if (middleNum == target) {
            return middle;
        }
        else if (middleNum > target) {
            // look in the left part
            h = middle - 1;
        }
        else {
            // look in the right part
            l = middle + 1;
        }

    }

    // not found, and now l == h + 1
    return l;
}
