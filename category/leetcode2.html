201-bitwise_and_of_numbers_range

Jan 25 2016 Tags AND / ANDing Consecutive Numbers

LeetCode Problem Link

The straightforward linear time approach will give you TLE. Here is a very clever solution based on the observation that ANDing consecutive non-negative integers will wipe out the lower bits and keep only the bits that are set both in m and n.

public int rangeBitwiseAnd(int m, int n) {
    int cnt = 0;

    while (m != n) {
        m = m >> 1;
        n = n >> 1;
        cnt++;
    }

    return (m <<cnt);
}

We could do it bit by bit. ANDing the current number with 1 gives us the least significant bit.

public int reverseBits(int n) {
    int num = 0;

    for (int i=0; i<32; i++) {
        int b = n & 1;
        num = num | (b << (31-i));
        n = n>>1;
    }

    return num;
}

LeetCode Problem Link

The O(n) space solution is quite straightforward.

public void rotate(int[] nums, int k) {
    int n = nums.length;
    int[] newNums = new int[n];

    for (int i=0; i<n; i++) {
        int from = i-k;

        while (from < 0)
            from += n;

        newNums[i] = nums[from];
    }

    for (int i=0; i<n; i++) {
        nums[i] = newNums[i];
    }
}
