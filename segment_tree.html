segment_tree

Feb 28 2016 Tags Segment Tree

Segment tree is useful when you need to perform some range operations repeatedly.

For example, given an array of integers nums, find out the sum in the range [i,j].

A segment tree is a full binary tree. Each node either has 0 or 2 children. Each node in the segment tree represents a value in some range. For our specific problem, it would represent a sum of all the values in that range. The range of leaf nodes are single-valued range (i.e., [i]) So for an input array nums of size n, our segment tree will have exactly n leaf nodes.

Given this information, we could actually find out how many total nodes will be in our segment tree.

Some other sample problems:

Given an array of integers nums, find out the min in the range [i,j]. Given an array of integers nums, find out the max in the range [i,j].

When you are doing either update of range query, remember that there are 3 cases - no overlap (stop looking further) - complete overlap (stop looking further) - partial overlap (need to recurse into the children nodes)

building the tree takes O(n) time update() takes O(logn) time query() takes O(n) time
