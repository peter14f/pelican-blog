494-target_sum

Mar 17 2018 Tags Dynamic Programming / Recursion / DFS / Knapsack Problem / 0-1 Knapsack Problem

LeetCode Problem Link

The recursive solution is not hard to come up, but it takes exponential time.

public int findTargetSumWays(int[] nums, int S) {
    // 2 ^ n total possible choices
    return numWaysSumToTarget(nums, 0, 0, S);
}

private int numWaysSumToTarget(int[] nums, int i, int curSum, int target) {
    if (i >= nums.length) {
        if (curSum == target) {
            return 1;
        } else {
            return 0;
        }
    } else {
        int a = numWaysSumToTarget(nums, i+1, curSum+nums[i], target);
        int b = numWaysSumToTarget(nums, i+1, curSum-nums[i], target);
        return a + b;
    }
}

Recursive (DFS) solution. 1D array can also be thought of as a graph. Each element only has one successor, the element to the right. The last element has no successor. This approach takes O(2^n) time.

public boolean canPartition(int[] nums) {
    int totalSum = 0;

    for (int num : nums) {
        totalSum += num;
    }

    if (totalSum % 2 == 1) {
        return false;
    }

    return canPartitionIntoTwoEqualSum(nums, 0, 0, totalSum);
}
