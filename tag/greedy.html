406-queue_reconstruction

LeetCode Problem Link

public int[][] reconstructQueue2(int[][] people) {
    int n = people.length;
    int[][] ans = new int[n][];

    PriorityQueue<int[]> pq = new PriorityQueue<>(new MyComparator());

    for (int[] entry : people) {
        pq.offer(entry);
    }

    LinkedList<int[]> ll = new LinkedList<>();
    while (!pq.isEmpty()) {
        int[] person = pq.poll();
        ll.add(person[1], person);
    }

    Iterator<int[]> itr = ll.iterator();
    int i=0;
    while (itr.hasNext()) {
        ans[i] = itr.next();
        i++;
    }

    return ans;
}

dijkstra_algorithm

Mar 02 2016 Tags Dijkstra / Greedy / Graph / Weighted Graph / Undirected Graph / Directed Graph / Hashheap

In most pseudocode you will find for Dijkstra, they will say initialize the Q such that all nodes are put in the Q.

The problem with this approach in Java is that when you update the distance to the node, PrioirtyQueue does not provide an API where you can update an item's priority. So you must remove that element and then re-insert it again. Note that PQ.remove() will that O(n) time. If you want it to take only O(logn) time you will have to implement a Hashheap yourself to achieve the O(logn) time removal.
