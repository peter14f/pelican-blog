004 Median of Two Sorted Arrays

Aug 30 2015 Tags Sorted Arrays / Merge Sort / Binary Search

LeetCode Problem Link

Let's forget about the time constraint here. How would you approach the problem?

We can use a similar approach when we're doing a merge sort. Use two pointers i and j.

(i+j) is the number of times the pointers have been incremented.

And min(nums1[i], num2[j]) is the (i+j+1)th element.

i=0, j=0 1st element is min(nums[i], nums[j])

public double findMedianSortedArrays(int[] nums1, int[] nums2) {
    int m = nums1.length;
    int n = nums2.length;

    if ((m + n) % 2 == 0) {
        // even number of total elements

        int k = (m+n)/2;

        double a = findKthElement(nums1, nums2, k);
        double b = findKthElement(nums1, nums2, k+1);

        return (a+b)/2;
    } else {
        // odd number of total elements
        int k = (m + n)/2 + 1;
        double ans = findKthElement(nums1, nums2, k);
        return ans;
    }
}

private double findKthElement(int[] nums1, int[] nums2, int k) {
    int i=0;
    int j=0;

    while (i+j+1 < k) {
        if (i >= nums1.length) {
            j++;
        } else if (j >= nums2.length) {
            i++;
        } else {
            if (nums1[i] <= nums2[j]) {
                i++;
            } else {
                j++;
            }
        }
    }

    if (i >= nums1.length) {
        return nums2[j];
    } else if (j >= nums2.length) {
        return nums1[i];
    }

    if (nums1[i] <= nums2[j]) {
        return nums1[i];
    } else {
        return nums2[j];
    }
}

This is the O(m+n) time solution.

We are asked to do this in O(log(m+n)) time where m is the size of the first sorted array and n is the size of the second sorted array.

If (m+n) is odd, we need to find the (m+n)/2 + 1 th smallest element.

Otherwise, we need to find (m+n)/2 th smallest element and (m+n)/2 + 1 th smallest element and then take the mean.

Here is how we can get the K th element of the union of two sorted arrays A and B:

Suppose we choose the a th smallest element of array A, A[a-1],

and the b th smallest element of array B, B[b-1], such that a+b = K.

If A[a-1] == B[b-1], then A[a-1] is the K th smallest element of the union of A and B.

We're done!

If A[a-1] < B[b-1], then A[a-1] is the a th smallest element of the union of A and B.

Numbers in A with a index <= a-1 can be discarded

Numbers in B with a index > b can also be discard

We won't find the K th smallest element here in these two regions

If A[a-1] > B[b-1], then B[b-1] is the b th smallest element of the union of A and B.

Number is A with a index > a can be discarded

Numbers in B with a index <= b-1 can be discarded

We won't find the K th smallest element here in these two regions

The problem is how we choose a and b.

Here's one way we could do it, try to cover the small array while making sure the the other array has at least 1 element covered.

if (aSize <= bSize) {

    b = k - a;

else {
